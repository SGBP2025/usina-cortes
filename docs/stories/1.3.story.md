## Dev Notes

**Decisão Aria:** Usar **Supabase Auth nativo** com Server Actions do Next.js 14. NÃO usar bcrypt customizado — o Supabase Auth já faz hash da senha internamente.

**Server Actions vs Route Handlers:** Para auth, usar Server Actions (`"use server"`) pois simplificam o fluxo de cookies e redirect. Route Handlers (`/api/auth/`) são usados para webhooks externos (ex: Stripe).

**CRÍTICO Quinn:** O `UPLOAD_POST_TOKEN` do `server.js` original não será portado. A autenticação é feita 100% via Supabase Auth. Nenhum endpoint sem autenticação.

**Plano Starter padrão:** Criar via seed/migration o plano com `id = 'starter'`, 30 minutos inclusos, R$0,10/min adicional. O registro associa o novo usuário a este plano automaticamente.

**Error messages:** Traduzir os erros do Supabase Auth para português. Ex: "Email not confirmed" → "Confirme seu email antes de fazer login".

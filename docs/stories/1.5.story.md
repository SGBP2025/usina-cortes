## Dev Notes

**Decisão Aria:** O worker é um **processo Node.js persistente** no Railway, não uma função serverless. FFmpeg precisa de processos de longa duração (timeout de funções serverless seria problema).

**CRÍTICO:** Nunca commitar arquivos `/tmp/` ou `/downloads/` do `server.js` original. Adicionar ao `.gitignore`.

**Bull vs BullMQ:** Usar **Bull** (não BullMQ) por simplicidade no MVP. BullMQ é para quando precisar de múltiplos workers concorrentes.

**Retry automático do Bull:** Configurar `attempts: 3, backoff: { type: 'exponential', delay: 5000 }`. Se job falhar 3x, status='error' no Supabase.

**FFmpeg no macOS local (dev sem Docker):** Pode ser instalado via `brew install ffmpeg`. No `ffmpeg.ts`, tentar: `which ffmpeg` → `/opt/homebrew/bin/ffmpeg` → `/usr/local/bin/ffmpeg` → `ffmpeg` (PATH).

**Créditos:** O débito de créditos ocorre ANTES de enfileirar (validado na Story 2.3). O worker apenas registra o `credits_consumed` real após processar.

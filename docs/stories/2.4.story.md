# Story EPIC-001.10: Upload de Vídeo

## Metadata
- **Story ID:** EPIC-001.10
- **Epic:** EPIC-001 — Transformação SaaS
- **Sprint:** 2 — Core Features
- **Points:** 8
- **Status:** Done
- **Priority:** P0
- **Depende de:** EPIC-001.2, EPIC-001.3, EPIC-001.4

## Story
**As a** usuário autenticado  
**I want** fazer upload de um vídeo MP4  
**So that** o sistema possa processá-lo e gerar clipes virais

## Acceptance Criteria
- [ ] AC1: Página `/dashboard/upload` com drag-and-drop zone
- [ ] AC2: Aceitar apenas MP4/MOV/AVI, máximo 500MB
- [ ] AC3: Progress bar durante upload para Supabase Storage
- [ ] AC4: Ao concluir upload: INSERT em `video_files` + INSERT em `processing_jobs` (status=pending) + enqueueVideoJob()
- [ ] AC5: Redirecionar para `/dashboard/jobs/[id]` após enfileirar
- [ ] AC6: Verificar créditos disponíveis antes de aceitar o upload (mínimo 5 min)
- [ ] AC7: Exibir erro claro se créditos insuficientes

## Tasks
- [ ] T1: Criar `app/(dashboard)/upload/page.tsx` (client component)
- [ ] T2: Criar `app/api/jobs/create/route.ts` — API Route que recebe jobId+fileId e enfileira
- [ ] T3: Criar `components/molecules/DropZone.tsx` com drag-and-drop
- [ ] T4: Criar `components/molecules/UploadProgress.tsx`
- [ ] T5: Lógica de upload direto para Supabase Storage (client-side, sem proxy)

## Dev Notes
Upload direto ao Supabase Storage via `supabase.storage.from('videos').upload()` no cliente — não passa pelo Next.js para evitar limite de 4MB de body. O API route `/api/jobs/create` só recebe o `video_file_id` após upload concluído.

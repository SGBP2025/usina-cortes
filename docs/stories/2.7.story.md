# Story EPIC-001.13: Tracking de Uso em Tempo Real

## Metadata
- **Story ID:** EPIC-001.13
- **Epic:** EPIC-001 — Transformação SaaS
- **Sprint:** 2 — Core Features
- **Points:** 5
- **Status:** Done
- **Priority:** P1
- **Depende de:** EPIC-001.11, EPIC-001.12, EPIC-001.2

## Story
**As a** usuário  
**I want** ver o status em tempo real do job sendo processado  
**So that** saiba quando meus clipes estarão prontos

## Acceptance Criteria
- [ ] AC1: Página `/dashboard/jobs/[id]` exibe status atualizado (pending → processing → completed/error)
- [ ] AC2: Polling automático a cada 3s enquanto status = pending/processing
- [ ] AC3: Progress steps visuais: Upload → Áudio → Transcrição → IA → Corte → Concluído
- [ ] AC4: Ao status = completed: exibir lista de clipes gerados com botão de download
- [ ] AC5: Ao status = error: exibir mensagem de erro e botão "Tentar novamente"
- [ ] AC6: Cancelar polling quando status = completed ou error

## Tasks
- [ ] T1: Atualizar `app/(dashboard)/jobs/[id]/page.tsx` com polling
- [ ] T2: Criar `app/api/jobs/[id]/status/route.ts` — retorna status atual do job
- [ ] T3: Criar `components/organisms/JobStatusView.tsx` com steps visuais
- [ ] T4: Criar `components/molecules/ClipCard.tsx` para exibir clipes gerados

## Dev Notes
Usar polling simples com `setInterval` no cliente (não Supabase Realtime — evitar complexidade). Cancelar no `useEffect` cleanup. O API route busca o job + clips via service role para garantir acesso.

## Story
**As a** time de desenvolvimento  
**I want** um pipeline CI/CD que valide código automaticamente e faça deploy no merge para main  
**So that** bugs sejam detectados antes de chegar em produção e deploys sejam automáticos

## Dev Notes

**Decisão Aria:** 
- **Vercel** para o Next.js (frontend + API routes) — free tier, edge network, integração nativa
- **Railway** para o worker — processo persistente, suporte FFmpeg, ~$5/mês no plano Hobby

**Branching strategy:** `main` (produção) + `develop` (staging) + `feature/X.Y-story-name` (desenvolvimento). PRs de feature → develop. PRs de develop → main (release).

**Railway vs Dockerfile:** O Railway pode usar o `worker/Dockerfile` diretamente. Configurar em `railway.toml`:
```toml
[build]
dockerfilePath = "worker/Dockerfile"
```

**Variáveis de ambiente de produção:** NÃO commitar em nenhum arquivo. Usar GitHub Secrets para o CI/CD e Railway/Vercel dashboard para as variáveis de runtime.

**Testes no CI:** Por enquanto apenas lint + build. Os testes Jest/Playwright serão adicionados no Sprint 5 conforme Quinn definiu. Preparar o placeholder: `npm test -- --passWithNoTests`.

## Testing

- Teste de CI: PR com `console.log` na codebase → ESLint deve bloquear se configurado com `no-console: error`
- Teste de deploy: push para main → verificar logs do Railway e Vercel para deploy bem-sucedido
